# Desktop GUI Visual Direction

This note is for future desktop-shell research, not for redefining the current product identity.

## Product identity stays the same

`sg-preflight` remains:

- SG QA
- 3D Car preflight
- deterministic checks
- `.pdx/checkers` awareness
- evidence and handoff tooling
- honest blocker visibility

The visual references below are for a future desktop operator shell only.
The current experimental desktop shell already uses a first translated pass of that language, but the product framing still stays SG-native.

## Desired qualities

- high-clarity task hierarchy
- strong local-tool energy without parody
- dark structured containers
- green activity / scanline accents
- amber emphasis for section titles or warnings
- deliberate button-guide rhythm
- stronger local-tool framing than a generic internal dashboard

## Current translated pass

Desktop Operator Shell v0 now translates these qualities into the live Qt shell through:

- scanline header bars with amber title emphasis
- compact category-tab action strip
- grid-framed dark panels with green edge activity
- TV-static-style evidence framing for the `Open first` panel
- bottom button-guide controls instead of a generic flat toolbar

That is intentional reference translation, not an identity swap.

## Guardrails

- do not turn the product into game cosplay
- do not let references replace SG-side workflow clarity
- do not let visual styling hide blockers or workflow boundaries
- keep the evidence/handoff purpose more important than decoration
