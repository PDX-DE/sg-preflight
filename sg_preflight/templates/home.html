Why This Should Save Time

These are the retro-driven promises the UI needs to satisfy in practice.

Catch avoidable findings earlier
Use the live profiles before rack or preview delivery instead of discovering obvious issues at the end.
Reduce source-of-truth confusion
Every run links back to the SG source files that produced the finding, instead of forcing manual rediscovery.
Make ownership and handoff clearer
Grouped findings, owner hints, action hints, and markdown evidence are meant to make the next step obvious.
